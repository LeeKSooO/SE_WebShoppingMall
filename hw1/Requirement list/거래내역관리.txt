// 이한슬

// 상품 구매 내역 관리 기능
[1] 
Requrement: 회원은 구매 내역(상품명, 제작회사명, 판매자, 가격, 평균 구매만족도, 구매일)을 조회할 수 있다.
Use Case: 구매 내역 조회
Func/Non-Func: Functional
액터: 회원

[2]
Requrement: 이때 상품명의 오름차순으로 정렬해서 출력한다.
Use Case: 구매 내역 오름차순 정렬 (extend 구매내역 조회)
Functional
액터: 회원

[3]
Requrement: 조회된 구매 내역 중에서 하나의 상품을 선택하여 구매만족도를 평가할 수 있다.
Use Case: 구매 만족도 평가
Func/Non-Func: Functional
액터: 회원

[4]
Requrement: 조회된 구매 내역 중에서 하나의 상품을 선택하여 구입 후 4주가 지나지 않은 상품의 경우 환불을 신청할 수 있다.
Use Case: 환불 신청
Func/Non-Func: Functional
액터: 회원
[5]
Requrement: 환불을 신청하는 경우 택배사에 자동으로 물품 수거 신청이 요청된다
Use Case: 택배사에 물품 수거 신청
Func/Non-Func: Functional
Event: 환불 신청
외부 시스템: 택배사

// 상품 구매 내역 삭제 기능
[1]
Requrement: 3개월이 지난 구매 거래내역부터 임의로 삭제가 가능하며 그 이전의 거래내역은 임의로 삭제가 불가능하다.
Use Case: 구매 내역 삭제
Func/Non-Func: Functional
액터: 회원

[2]
Requrement: 6개월이 지난 거래내역은 자동으로 삭제된다.
Use Case: 구매 내역 삭제
Func/Non-Func: Functional
Event: 거래 내역 6개월 지남

// 상품 판매/구매 통계 기능
[1]
Requrement: 회원이 판매한 상품에 대한 총액 및 평균 구매만족도와 구매한 상품에 대한 총액 및 평균 구매만족도를 출력한다. (삭제된 구매 내역은 통계에서 제외한다)
Use Case: 판매/구매 상품의 총액과  평균 구매 만족도 출력
Func/Non-Func: Functional
액터: 회원

[2]
Requrement: 매월 말일에 모든 회원들에게 그 달에 대한 판매 및 구매 통계 정보를 이메일로 공지된다
Use Case: 통계 정보 통지
Func/Non-Func: Functional
Event: 매월 말일
외부 시스템: 이메일 시스템
