// 이한슬

[1]
Actor: 내역 관리자
Use Cases: 판매 완료 내역 조회, 구매 내역 조회, 구매 만족도 평가, 환불 신청, 구매 내역 삭제, 판매/구매 상품 총액과 평균 구매 만족도 출력
Description: 내역 관리자는 회원이라는 계정과 관련된 역할에서 분리한 역할로, 거래 내역에 관한 기능들과 관련 있는 액터이다. 시스템에서 제공하는 판매 내역 조회, 구매 내역 조회, 구매 만족도 평가, 환불 신청, 구매 내역 삭제, 판매/구매 상품 총액과 평균 구매 만족도 출력 기능들을 사용할 수 있다.

[2]
Actor(Event): 판매 종료일 지남
Use Cases: 판매 완료 기능
Description: 판매 종료일이 지난 상품은 판매 완료 상태가 되어 판매 완료 내역에 나타난다.

[3]
Actor(Event): 남은 수량이 0이 됨
Use Cases: 판매 완료 기능
Description: 남은 수량이 0이 된 상품은 판매 완료 상태가 되어 판매 완료 내역에 나타난다.

[4]
Actor(Event): 환불 신청
Use Cases: 물품 수거 신청
Description: 회원이 어떤 구매 상품에 대하여 환불 신청을 하는 Event가 발생하면 자동으로 물품 수거 신청이 요청된다.

[5]
Actor: 택배사(외부 시스템)
Use Cases: 물품 수거 신청
Description: 회원이 환불 신청을 하는 Event가 일어나면 외부 시스템인 해당 상품을 배송하는 택배사로 물품 수거 신청이 요청된다.

[6]
Actor(Event): 거래 내역 6개월 지남
Use Cases: 구매 내역 삭제
Description: 거래 내역이 6개월이 지나면 자동으로 삭제된다.

[7]
Actor(Event): 매월 말일
Use Cases: 통계 정보 통지
Description: 매월 말일에 모든 회원들에게 그 달에 대한 판매 및 구매 통계 정보가 공지된다.

[8]
Actor: 이메일 시스템(외부 시스템)
Use Cases: 통계 정보 통지
Description: 매월 말일, 회원들에게 통계 정보가 공지될 때, 외부 시스템인 이메일 시스템을 통해 각 회원들에게 공지가 발송된다.
